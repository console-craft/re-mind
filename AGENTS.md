# re-mind

Voice-first reminder app built with a React SPA frontend and a FastAPI backend.

## Using Vite+, the Unified Toolchain for the Web

The frontend in `web/` uses Vite+, a unified toolchain built on top of Vite, Rolldown, Vitest, tsdown, Oxlint, Oxfmt, and Vite Task. Vite+ wraps runtime management, package management, and frontend tooling in a single global CLI called `vp`. Vite+ is distinct from Vite, and it invokes Vite through `vp dev` and `vp build`. Run `vp help` to print a list of commands and `vp <command> --help` for information about a specific command.

Docs are local at `web/node_modules/vite-plus/docs` or online at https://viteplus.dev/guide/.

## Coding Style and Conventions

- Keep files <= ~200 LOC; functions <= ~50 LOC. Split larger files by concern rather than extracting utilities.
- Exception: test files in `tests/` can exceed ~200 LOC when keeping related assertions together improves test clarity and signal.
- Add JSDoc comments for TypeScript helper functions with a description, parameters, and return type. Keep them in sync with the actual code.
- Add Python docstrings for public modules, helpers, endpoints, and tests.
- Some duplication is ok for clarity and to avoid over-abstraction. Too much duplication is bad.
- Prefer explicit return types for functions.
- Don't use single line blocks (`if (x) return y;`); prefer braces/newlines for clarity.
- Prefer function declarations over function expressions, except when a function expression improves readability.
- Use plain CSS and CSS variables for the frontend. Do not add Tailwind or shadcn UI unless explicitly requested.
- The frontend is dark-mode only for now.
- Use `web/src/features/` for frontend domain feature code.
- Use feature `helpers.ts` files for feature-specific pure logic. Use `web/src/lib/utils.ts` only for generic cross-feature helpers.

## Verification

- Before marking work as complete, if code files or project config files were added or changed, run the `/verify` command or the equivalent `just check` command.

## Repo Map

- `web/` contains the React SPA, generated by Vite+ and managed with pnpm.
- `web/src/main.tsx` mounts the React app and imports global styles.
- `web/src/App.tsx` composes the top-level frontend shell.
- `web/src/index.css` contains global dark-only CSS variables and base styles.
- `web/src/App.css` contains the first app-shell component styles.
- `web/src/components/` contains React components that render the app UI.
- `web/src/features/` contains frontend domain features.
- `web/src/features/reminders/` contains reminder feature code: API calls, type aliases, and feature-specific helpers.
- `web/src/features/voice/` contains Vapi browser call state and voice-assistant date/time helpers.
- `web/src/lib/api-client.generated.ts` is reserved for generated OpenAPI types/client output.
- `web/src/lib/utils.ts` is reserved for generic cross-feature frontend utilities.
- `api/` contains the FastAPI backend, managed with uv.
- `api/app/main.py` defines the FastAPI app, local CORS setup, and mounted feature routers.
- `api/app/database.py` contains SQLite path, connection, and schema initialization helpers.
- `api/app/events/` contains the in-memory server-sent event broadcaster and `/events` stream route.
- `api/app/reminders/` contains the reminder backend feature: router, schemas, and repository.
- `api/app/vapi/` contains Vapi webhook schemas, secret validation, routing, and reminder tool handlers.
- `api/tests/` contains backend tests for health, reminders, and Vapi webhook behavior.
- `data/` is reserved for local SQLite data and should not contain committed database files.
- `vapi-system-prompt.md` and `vapi-tool-descriptions.md` document Vapi assistant prompt and tool configuration.
- `.env.example` documents local environment variables for the API, frontend, and Vapi integration.
- `justfile` contains local development shortcuts.
- `.opencode/commands/verify.md`, `.opencode/agents/verify.md`, and `.opencode/skills/check/SKILL.md` define the local verification workflow.
